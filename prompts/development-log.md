# 增量记录开发日志

当用户说“写日志”“记录开发日志”“增量记录开发日志”或同语义表达时，执行以下操作：

1. 读取当前仓库中的开发日志文件，优先使用 `docs/development-log.md`。
2. 在日志末尾追加一条增量记录，不覆盖、不重写已有日志。
3. 记录本轮相关上下文，包括：
   - 用户提出的关键原话，使用 `👤 用户：` 标注。
   - 助手给出的关键结论或执行结果，使用 `🤖 助手：` 标注。
   - 实际完成的代码、文档、架构或命令变更。
   - 验证过的命令和结果摘要。
   - Git 提交 hash、提交信息、push 状态、工作区状态，如果本轮涉及 Git。
4. 日志格式保持 Markdown，按时间追加二级标题：
   `## YYYY-MM-DD：简短主题`
5. 如果用户要求“原文记录”，保留用户原话和助手原文，不做摘要改写。
6. 如果只是普通“记录开发日志”，可以简洁摘要，但必须保留关键事实、文件路径、命令和结果。
7. 写完后检查 `git status --short`，并告知用户当前有哪些文件发生变化。
8. 不自动提交，除非用户明确要求提交。
